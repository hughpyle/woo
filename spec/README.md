# Woo Specification

This directory holds the Woo specification. It should become detailed enough to
guide implementation, testing, operations, and future compatibility decisions.

## Contents

- [Vision](vision.md): what Woo is trying to become, and what it is not yet.

## Open Specification Areas

- Object model
- World model
- Language and programmability
- Security and permissions
- Persistence and history
- Networking and client protocol
- Identity and presence
- Administration and governance
- Import or compatibility story for LambdaMOO concepts
- Deployment architecture
- Testing and conformance

## Process

The spec comes first. Implementation work should begin only after the relevant
section of the spec is explicit enough to constrain what is being built.
